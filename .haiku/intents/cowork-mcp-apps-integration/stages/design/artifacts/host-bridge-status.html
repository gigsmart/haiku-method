aria-live announcement strings — HostBridgeStatus
trigger: component mount (any state) "MCP Apps: [state label]" region: aria-live="polite" aria-atomic="true" on the pill/button
trigger: state → connected (on mount or reconnect recovery) "MCP Apps: Connected" region: aria-live="polite"
trigger: state → reconnecting (bridge heartbeat dropped) "MCP Apps: Reconnecting" region: aria-live="polite"
trigger: state → error (bridge failed, not auto-recovering) "MCP Apps: Connection error – click to retry" region: aria-live="polite" aria-atomic="true" on the button
trigger: user clicks error pill → retry in flight "Retrying connection…" region: aria-live="polite" (pill transitions to retry-pending state)
trigger: retry succeeded "MCP Apps: Connected" region: aria-live="polite"
trigger: retry failed "Retry failed. Click to try again." region: aria-live="assertive" on retry failure only (higher urgency)
Implementation note: A single visually hidden <span aria-live="polite" aria-atomic="true"> adjacent to the topbar is sufficient for all polite announcements. The retry-failure announcement uses aria-live="assertive" because the user needs to know immediately that their action failed. All announcement strings are injected into the live region via JS text replacement, not via aria-label mutation alone, to guarantee NVDA/JAWS compatibility.
