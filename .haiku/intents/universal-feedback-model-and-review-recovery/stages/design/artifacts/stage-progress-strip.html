Focus-order policy

Every stage node has an explicit tabindex. This eliminates ambiguity between "focusable-but-no-action" and "not in the tab order" — two states that look identical to pointer users but are wildly different for keyboard users.

Stage condition tabindex Hover Focus ring Activation
Completed (prior stage) 0 teal border + tooltip teal 2px outline, 2px offset Enter opens stage (jump-to-stage)
Current (in-progress) 0 diamond badge lifts teal outline on diamond Enter scrolls to stage's active unit
Previously visited (now "future") 0 border darkens, tooltip teal outline Enter opens read-only view of prior visit
Future, never visited -1 tooltip only (no border change) not focusable via Tab n/a — no action

Future never-visited stages remain in the a11y tree (role="link" + aria-disabled="true") so assistive tech still announces their existence — they are just not a Tab stop, because there is nothing to activate.
