Where this card renders

The assessor summary lives in the review-sidebar footer (sticky, just above the user-gate bar), not as a dedicated banner. Rationale:

  • It's a sibling of the feedback list — the reviewer scans the list, then glances at the roll-up.
  • Banners compete for the same space as the rollback-reason banner (different state, same surface = wrong).
  • Sticky footer keeps the numeric summary in view while the reviewer scrolls the feedback list.

Interactive states · touch targets · a11y

States spec

  • default — status dot (green / amber / rose) + pill, roll-up counts, collapsed bullet list.
  • hover — bullet list row background hover:bg-gray-50 dark:hover:bg-gray-800/60; cursor pointer if linked to feedback item.
  • focus — focus ring focus:ring-2 focus:ring-teal-500 on any interactive element (view details button, per-item rows).
  • active — button pressed state via active:bg-blue-900/60.
  • disabled — view-details hidden until assessor pass completes (no "view details" in error state with empty log).
  • loading — card renders with skeleton stat row + spinner animate-spin on status dot, pill reads running.
  • empty — 0 feedback items: card hides entirely (no empty-state copy needed; stage is clean by definition).

Accessibility

  • Card is a live region — role="status" aria-live="polite" — so screen readers pick up "pass clean" / "3 pending" when the assessor finishes.
  • Status dot has aria-hidden="true"; color is never the only signal — the adjacent pill text reads clean / pending / error.
  • Contrast: amber-800 on amber-50 = 7.4:1, rose-800 on rose-50 = 7.1:1, green-700 on white = 5.4:1. All AA-compliant for body text.
  • Touch targets on interactive buttons (view details, view log) meet 44×44 via px-2 py-1 + 8px hit-area extension (cursor target = entire row).
  • Keyboard: Tab focuses view-details → per-item rows (if expanded). Enter / Space activates. Esc collapses expanded list.

Responsive · breakpoints

  • mobile (375px) — sidebar is bottom sheet; assessor card becomes the sticky sheet header. Stat row stacks 3×1 → remains 3 columns (width fits), bullet list collapses by default, "view details" opens modal.
  • tablet (768px) — sidebar + card widths match existing ReviewSidebar tokens (w-80). No layout change.
  • desktop (1280px) — sidebar lg:w-96, bullet list expanded by default up to 4 items + scroll.

Tokens used (knowledge/DESIGN-TOKENS.md)

  • feedback-status-pending, feedback-status-addressed, feedback-status-closed, feedback-status-rejected (status pills, dots)
  • card — p-3 rounded-lg border (standard), shadow-sm (sidebar footer elevation)
  • stat tile — rounded-md, bg-gray-800/60 or bg-white border-gray-200
  • error surface — bg-rose-900/10 / bg-rose-50 + border-rose-300 / border-rose-800
  • pending surface — bg-amber-900/10 / bg-amber-50 + border-amber-300 / border-amber-700
  • focus ring — focus:ring-2 focus:ring-teal-500 (shared with ReviewSidebar)
  • per-item dots — bg-blue-500 / bg-blue-400, bg-amber-500 / bg-amber-400, bg-gray-400 (rejected/closed-prior)
