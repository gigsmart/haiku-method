Every visual state for the creation popover

Companion to annotation-gesture-spec.html. This file renders the popover in each distinct state so development can match pixel-for-pixel without guessing.

Popover dimensions are fixed at 288px × auto in desktop/tablet (matches current w-72 in the reference mockup). On small viewports (< 480px) the popover promotes to a bottom-sheet that spans full width.

State 1

Open — spatial anchor — empty fields

Reviewer has just clicked on an image/SVG. The popover anchors to the right-and-below of the click (offset 10px) unless it would clip the viewport edge, in which case it flips. Title input is auto-focused.

State 4

Error — save failed — retry inline

If the haiku_feedback call errors (network blip, validation, 500), the popover stays open with the captured fields preserved and a red inline banner at the top. No toast — the reviewer's attention is already in the popover.

State 5

Small viewport — bottom-sheet promotion

Below 480px the popover promotes to a bottom sheet: slides up from the bottom, full width, 16px top corner radius. Footer buttons grow to 44×44px touch targets. Title + body share the same surface; no layout change beyond size.

Review · Stage 3 FB: 4 pending

New feedback @ 38%, 52%

Touch targets

44×44px minimum per iOS/Material. Bottom sheet buttons are h-11 (44px) with text-sm.

Drag-to-dismiss

Swipe down on the drag handle to cancel. Prevents accidental keyboard-dismiss from closing the sheet on mobile.

Breakpoint

Sheet activates below 480px (just under the sm: Tailwind breakpoint). Above that, the popover stays anchored.

State 6

Dark mode parity

Forced-dark render so dark-mode color contrast is reviewable regardless of the current theme toggle. All other states above already ship dark variants via dark: utilities.

Token provenance — all values above

Every color, spacing, and typography value comes from the intent's knowledge/DESIGN-TOKENS.md. No raw hex values. No magic numbers.

  • Popover shell — rounded-xl border border-stone-200 dark:border-stone-700 shadow-2xl (matches existing Modal overlay token).
  • Popover border accent — border-teal-300 dark:border-teal-800 (accent, applied only when open).
  • Title / body input — text-xs p-2 border border-stone-300 dark:border-stone-600 rounded-md bg-white dark:bg-stone-800 focus:ring-2 focus:ring-teal-500 (standard Input token).
  • Create button — primary teal token (bg-teal-600 hover:bg-teal-700).
  • Cancel button — secondary outline token.
  • Ghost pin color — rgba(13, 148, 136, 0.6) == teal-600 / 60.
  • Bottom sheet touch targets — h-11 (44px) min, text-sm per mobile-breakpoint discipline.
  • Error banner — bg-red-50 border-red-200 text-red-700 (existing semantic Error token).
