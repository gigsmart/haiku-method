SSR templates cannot use React components directly. The shared package exports static HTML renderer functions alongside the React components for SSR consumption.

4. Styling Strategy

Source-only CSS distribution with a shared Tailwind preset for both consumers.

1

Source-Only

@haiku/review-ui ships uncompiled TSX/TS source. No built CSS, no compiled JS. The consumer's build pipeline (Vite for SPA, tsc for SSR) processes everything.

// package.json
{
  "exports": {
    ".": "./src/index.ts",
    "./ssr": "./src/ssr/index.ts"
  }
}
2

Shared Tailwind Preset

A single tailwind-review preset defines all feedback tokens. Both consumers extend it.

SPA uses stone neutral + teal accent
SSR uses gray neutral + blue accent
Feedback colors (amber, blue, green, rose) same in both
3

Dark Mode

All shared components use dark: variant classes. The consumer controls the .dark class on <html>.

Pattern for every color token:

Light: bg-{c}-100 Dark: bg-{c}-900/30
Light: text-{c}-800 Dark: text-{c}-300

  /** Ordered list of stages in the studio */

6. Package Configuration

packages/review-ui/package.json

{
  "name": "@haiku/review-ui",
  "version": "0.0.0",
  "private": true,
  "type": "module",
  "exports": {
    ".": "./src/index.ts",
    "./ssr": "./src/ssr/index.ts",
    "./presets": "./src/presets/tailwind-review.ts"
  },
  "peerDependencies": {
    "react": "^19.0.0"
  },
  "devDependencies": {
    "@types/react": "^19.1.2",
    "typescript": "^5.7.0"
  }
}

Zero runtime dependencies. React is a peer dep. Tailwind is not a dependency -- consumers bring their own. The "./ssr" subpath exports HTML string renderers for the SSR template path.

7. Architecture Flow

How data flows through the unified package in both the SPA and SSR paths.

SPA Path (React)

Browser → App.tsx
router → ReviewPage
state lifting → ReviewSidebar
composition → @haiku/review-ui
FeedbackList, StageProgressStrip, etc.

SSR Path (HTML Strings)

HTTP Request → http.ts
route handler → intent-review.ts
template → components.ts
render functions → @haiku/review-ui/ssr
renderFeedbackStatusBadge(), etc.

Shared Token Layer

Both paths import color maps, status color lookups, and icon maps from @haiku/review-ui. The SPA uses these as Tailwind class strings in JSX; the SSR path interpolates them into HTML template strings. Tokens are defined once, consumed everywhere.
