Comments List with Agent-Feedback Toggle

Unit 05 · FB-02 · Sidebar restructure — one unified Comments list, agent-feedback overlay via toggle · No "mine vs others"

1. Why no "Mine / Feedback" segmented control

  • H·AI·K·U has no concept of user identity. There is no login, no per-user state — the review app is a local surface. "Mine" is undefined.
  • User-origin items (user-chat, user-visual, external-pr, external-mr) unify into a single Comments list. Origin badges differentiate, so nothing is hidden.
  • Agent-origin items (adversarial-review, agent) are secondary — the reviewer rarely needs them during active review. An agent-feedback toggle (default OFF) reveals them inline with visible origin badges.
  • When the toggle is ON, agent items render inline with user items in the same list — interleaved by stage, not pushed to a separate tab. The reviewer still sees one list; the population just grows.

2. Sidebar — Light Mode, Toggle OFF (default)

Only user-origin items visible. Agent items are on disk but hidden; the toggle chip shows the muted count.

5. Toggle Interaction Spec

Behavior

  • Default OFF. First-time reviewers see only user-origin items so the list isn't overwhelmed with agent findings that haven't been triaged.
  • Persists per-intent in localStorage under haiku-review.toggle.agent-feedback.{intentSlug}. Not a global user setting — per-intent, because every review session has different noise characteristics.
  • Counts update live: the chip on the toggle always shows the opposite set ({n} hidden when off, {n} inline when on).
  • Keyboard shortcut: A inside the sidebar toggles it. ? lists shortcuts.
  • When switched on with active filter, newly-revealed items respect the current filter (status grouping, search query, etc.).

Accessibility

  • Implemented as a <button role="switch"> with aria-checked reflecting state.
  • Label and description both live in the DOM — description read via aria-describedby.
  • Focus ring on the toggle itself, 2px teal-500.
  • When toggled on/off, an aria-live="polite" region announces: "Agent feedback shown — 8 items added" or "Agent feedback hidden — 8 items removed".
  • Toggle retains focus after activation — no focus jump to the list below.

6. Rejected alternative — "Mine / Feedback" segmented control

What was in the earlier mockup

Two independent lists. User toggles between "my comments" and "everyone else's findings".

Why it's rejected

  • "Mine" is undefined. No user identity in H·AI·K·U. Every user-origin item is equally "mine" from any local session.
  • Hides useful context. When I'm deciding whether to dismiss my own pending comment, I may want to see a related review-agent finding on the same paragraph. Splitting them buries that.
  • Two lists = two focus targets. Every key action (Tab, arrow-down) now has to account for the current tab. A single list with an origin filter is simpler for keyboard users.
  • Doesn't match the task. The reviewer's task is "decide what to do with each comment", not "compare my output vs the agent's". Origin is metadata, not a primary axis.

7. Responsive Behavior

Mobile (375px)
  • Sidebar becomes a slide-in panel from the right, covering 85vw.
  • Toggle sticks to top; Comments list scrolls below.
  • Origin and status badges stack when the card width drops under 280px.
  • Toggle min-height 44px (touch target).
Tablet (768px)
  • Sidebar is 320px wide, always visible.
  • Cards show full badge row on one line at this width.
Desktop (1280px+)
  • Sidebar is 384px wide (lg:w-96).
  • Toggle shows long-form helper text under the switch.
  • Keyboard-nav annotations visible (A to toggle agent feedback).
