Complete Data Flow: Comment-to-Feedback Migration

Current Model (Batch)

User action (select/pin/type)

↓

React state (InlineCommentEntry[])

↓

Batch serialize on "Request Changes"

↓

POST /review/{id}/decide

{ decision, feedback: "long string...", annotations }

↓

Orchestrator receives ephemeral blob

⚠ No individual tracking, status, or lifecycle

New Model (Per-Item CRUD)

User action (select/pin/type)

↓

React state (unchanged)

↓

On "Request Changes", for each comment:

↓

POST /api/feedback/{intent}/{stage}

{ title, body, origin, source_ref }

→ feedback file on disk (status: pending)

↓

After all files created:

POST /review/{id}/decide

{ decision, feedback: "Created 4 items: FB-01..04" }

✓ Each item individually trackable
