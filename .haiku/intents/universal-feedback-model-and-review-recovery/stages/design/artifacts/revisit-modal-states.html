Modal lifecycle

① Opening (first paint)

  • Backdrop fades in: opacity 0 → 1 over 150ms.
  • Dialog scales in: scale-95 → scale-100, opacity 0 → 1, 120ms.
  • Body scroll locked.
  • Focus lands on Cancel.
  • Reduced-motion users skip the transform; fade only.

② Confirm clicked

  • Confirm enters loading state (spinner + aria-busy=true).
  • Cancel disables during in-flight call.
  • Backdrop click + ESC are ignored while loading.
  • On success: modal closes, showToast('haiku_revisit invoked', …, 'revisit') fires.
  • On failure: see § Error state.

③ Closing

  • Backdrop fades out 120ms.
  • Dialog scales out: scale-100 → scale-95, 100ms.
  • Body scroll restored.
  • Focus returns to the element that triggered the modal.

Error state — revisit failed

If haiku_revisit errors (push rejected, worktree locked, network), the modal stays open with an inline error banner below the body. Confirm returns to the default label and can be retried.

Remote rejected the push — the intent branch has diverged. Pull, then retry, or run haiku_repair to reconcile.

  • Error banner uses tokens from DESIGN-TOKENS.md §1.8 Semantic / Error: bg-red-50 / border-red-200 / text-red-800.

  • Cancel remains functional — user can back out without retrying.

Failure-on-confirm — mid-commit rollback

Distinct from the in-modal error above: this is the state where the modal already closed (the confirm handler returned synchronously but the async haiku_revisit call failed mid-commit — worktree was half-mutated). Mirrors the feedback-card toast+revert pattern in feedback-card-states.html: the UI optimistically advanced, the tool rejected, we roll back state and surface a toast.

⚠

Revisit rolled back

Remote rejected the state push. No changes were committed. Review UI restored.

  • Mirrors feedback-card-states.html §5 Optimistic UI: advance optimistically, revert on tool rejection, surface toast.
  • Toast is dismissible but also auto-persists until the user acts — rollback info is too important to auto-dismiss after 3s.
  • Retry re-opens the modal with the same target; Open repair drops into haiku_repair.
  • Focus moves to the Retry button when the toast mounts. Escape dismisses. role="status" + aria-live="polite" ensure SR announcement.
  • Under prefers-reduced-motion: reduce the toast-in slide is disabled; the toast appears in place.

Motion & reduced-motion

Standard motion

  • Backdrop fade: 150ms ease-out on open, 120ms on close.
  • Dialog: scale-95 → scale-100 with opacity 0 → 1, 120ms ease-out.
  • Button hover transitions: transition-colors (150ms).
  • Loading spinner: 0.9s linear, infinite.
  • Focus ring appears instantly (no transition) — respects system behavior.

Reduced-motion

  • @media (prefers-reduced-motion: reduce)
  • Dialog skips the scale transform — fade only.
  • Loading spinner replaced with a static "busy" dot pulse at 1.5s (cadence stays announced but doesn't spin).
  • Confirm button loses translate-y-px active displacement.
