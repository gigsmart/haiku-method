4. Color-Blindness Robustness

Vision type Color conflict Second signal that disambiguates
protanopia / deuteranopia (red-green) amber vs green left border and badges clock glyph (pending) vs checkmark glyph (closed) — distinct shapes
tritanopia (blue-yellow) blue vs stone-gray left border arrow glyph (addressed) vs × cross glyph (rejected) + text prefix
monochromacy / grayscale all colors collapse to similar grays all four glyphs distinct (⏱ ↗ ✓ ×) + "Closed ·" / "Rejected ·" text prefix + strikethrough
screen-dimmed / bright sunlight light-mode backgrounds blow out border, glyph, and text prefix retain contrast independently of bg fill

5. Verification Checklist

  • Every card renders (1) a colored left border, (2) a shape glyph, and (3) a status badge with explicit text label.
  • Compact (truncated) card state still shows all three signals — glyph sits before the title so it is visible even after truncate clips the title mid-word.
  • Rejected state applies line-through decoration-stone-500 only to the title span — the "Rejected ·" prefix stays upright (not struck-through) so it remains unambiguous when the title truncates mid-word.
  • Closed state does not use opacity-70; rejected state does not use opacity-50. Both convey "finality" through muted background (bg-green-50/60 / bg-stone-100) plus the status badge label.
  • Light and dark modes both tested — contrast ratios for every text span ≥ 4.5:1 and for every border/glyph ≥ 3:1.
  • All glyphs are aria-hidden="true" — screen readers announce the status via the badge's aria-label="Status: …" and the text prefix on the title.
