Review: Feedback Model

Overview

Problem

The current review workflow treats feedback as ephemeral batch comments that vanish after submission. When a reviewer requests changes, the agent receives a single feedback string with no structured identity.

Solution

Introduce a universal feedback model where each piece of feedback is a first-class file on disk with its own identity, status lifecycle, origin tracking, and resolution metadata.

Success Criteria

  • Feedback items are individually trackable
  • Status lifecycle: pending → addressed → closed
  • Sidebar shows feedback across review cycles
