Focus Ring & Cross-Flash Visual Contract

The visual contract for keyboard focus in the review UI. Covers the persistent focus ring applied to feedback cards, stage stepper buttons, and tab buttons, plus the transient cross-flash animation driven by Enter. Companion doc: keyboard-shortcut-map.html.

1. Token inventory

All values are named Tailwind tokens from knowledge/DESIGN-TOKENS.md. No raw hex in component code — the values below live only in the CSS token layer of index.css.

Token Light Dark Usage
--focus-ring-color teal-500 (rgb 20 184 166) teal-400 (rgb 45 212 191) Persistent ring for j/k, [/], tab buttons.
--focus-ring-thickness 2px Matches sidebar focus (tight surfaces).
--focus-ring-offset 2px (1px on .fb-card) Keeps the ring outside the element edge; 1px on feedback cards because the card already has a 3px left status stripe.
--flash-color teal-500 · α 0.7 teal-400 · α 0.7 Box-shadow ring on Enter flash of the focused feedback card.
--flash-bg teal-500 · α 0.15 teal-400 · α 0.18 Fill on the target artifact row during the same flash.
--flash-duration 1400 ms Fixed; matches mockup. Both the card flash and the target fill run for the same duration.
--flash-easing cubic-bezier(0.2, 0, 0, 1) (Tailwind ease-out) Fast early, gentle tail — reads as "attention-then-settle".

3. Cross-flash animation (Enter)

Pressing Enter while a feedback card is focused runs two simultaneous animations: a ripple on the focused card (.fb-flash) and a fill on the referenced target row (.unit-flash). Duration and easing are identical so the pair reads as one gesture.

Duration

1400 ms for both flashes.

Easing

cubic-bezier(0.2, 0, 0, 1) — fast onset, long tail.

Re-triggers

If Enter is pressed again mid-animation, the classes are removed and re-added on the next frame (ensures animation restarts).

Reduced-motion variant

When prefers-reduced-motion: reduce is active, both .fb-flash and .unit-flash drop their animations / transitions (see CSS in §5). The static visual cue still fires: the focused card keeps its teal ring and the target row is scrolled into view. The user never loses the connection — only the motion is removed.

4. Contrast & accessibility

Ring color Background Contrast WCAG 2.2 non-text
teal-500 (#14b8a6) white (#ffffff) 3.02 : 1 passes (≥ 3 : 1)
teal-500 (#14b8a6) stone-50 (#fafaf9) 3.04 : 1 passes
teal-400 (#2dd4bf) stone-900 (#1c1917) 8.9 : 1 passes
teal-400 (#2dd4bf) stone-950 (#0c0a09) 9.4 : 1 passes
  • Per WCAG 2.2 SC 2.4.11 (Focus Appearance), the ring meets the non-text contrast minimum of 3 : 1 against every adjacent surface we render focus on.
  • Ring area (≥ 2 px × perimeter) exceeds the SC 2.4.11 minimum pixel-area requirement on every target.
  • The ring is never replaced by color alone — focused feedback cards additionally receive a visible border color shift and a screen-reader announcement (polite live region).
  • Focus moves are always programmatic + scrolled into view (scrollIntoView({ behavior: 'smooth', block: 'nearest' })) so users with low vision always have the focused target on-screen.

5. Ship-ready CSS (copy into index.css)

/* Tokens (names live here, values reference Tailwind scale) */
:root {
  --focus-ring-color: theme('colors.teal.500');
  --flash-color: rgb(from theme('colors.teal.500') r g b / 0.7);
  --flash-bg:    rgb(from theme('colors.teal.500') r g b / 0.15);
}
.dark {
  --focus-ring-color: theme('colors.teal.400');
  --flash-color: rgb(from theme('colors.teal.400') r g b / 0.7);
  --flash-bg:    rgb(from theme('colors.teal.400') r g b / 0.18);
}

/* Persistent focus ring (applied to stepper buttons, tab buttons, generic targets) */
.focus-target          { outline: 2px solid transparent; outline-offset: 2px; transition: outline-color 120ms ease; }
.focus-target.is-focused { outline-color: var(--focus-ring-color); }

/* Feedback card variant (tighter offset) */
.fb-card.is-focused { outline: 2px solid var(--focus-ring-color); outline-offset: 1px; }

/* Cross-flash */
@keyframes fbFlash {
  0%   { box-shadow: 0 0 0 0 var(--flash-color); }
  50%  { box-shadow: 0 0 0 6px rgba(0,0,0,0);     }
  100% { box-shadow: 0 0 0 0 rgba(0,0,0,0);       }
}
.fb-flash   { animation: fbFlash 1400ms cubic-bezier(0.2, 0, 0, 1); border-color: var(--focus-ring-color) !important; }
.unit-flash { background-color: var(--flash-bg) !important; border-color: var(--focus-ring-color) !important;
              transition: background-color 1400ms cubic-bezier(0.2, 0, 0, 1),
                          border-color     1400ms cubic-bezier(0.2, 0, 0, 1); }

@media (prefers-reduced-motion: reduce) {
  .fb-flash, .unit-flash { animation: none !important; transition: none !important; }
}

6. JS trigger contract

// On `Enter` while focusedFbId !== null:
function highlightFeedback(fbId) {
  const fbEl = document.querySelector(`.fb-card[data-fb-id="${fbId}"]`);
  const targetEl = document.querySelector(`[data-feedback-target="${fbId}"]`);
  if (!fbEl) return;

  // Restart trick: strip class, force reflow, re-add — so Enter-spam always re-plays the animation.
  fbEl.classList.remove('fb-flash');
  void fbEl.offsetWidth;
  fbEl.classList.add('fb-flash');

  if (targetEl) {
    targetEl.scrollIntoView({ behavior: 'smooth', block: 'nearest' });
    targetEl.classList.remove('unit-flash');
    void targetEl.offsetWidth;
    targetEl.classList.add('unit-flash');
    // Clean up after animation so DOM stays clean.
    setTimeout(() => targetEl.classList.remove('unit-flash'), 1500);
  }
  setTimeout(() => fbEl.classList.remove('fb-flash'), 1500);
}
