Review: Universal Feedback Model

Overview -- Comment on text

Select text to add inline comments.

Problem

The current review workflow treats feedback as ephemeral batch comments that vanish after submission. When a reviewer requests changes, the agent receives a single feedback string with no structured identity, status tracking, or lifecycle management.

This means feedback items cannot be individually tracked, addressed, or closed. Revisiting a stage after changes loses all context about which items were addressed and which remain open.

Solution

Introduce a universal feedback model where each piece of feedback is a first-class file on disk with its own identity, status lifecycle, origin tracking, and resolution metadata.

Each feedback item progresses through: pending → addressed/rejected → closed. The review app sidebar gains a dedicated feedback panel showing all items across review cycles.

Missing null check in handleSubmit callback

The handleSubmit function in ReviewSidebar.tsx:84 calls getAnnotations() without checking if the result is undefined. When the user submits with no comments and no pins, this passes undefined to submitDecision, which then serializes it as the string "undefined" in the feedback payload.

Add a guard: const annotations = getAnnotations() ?? {}
