Agent Feedback Toggle — Accessible Switch Spec

Closes FB-32. Replaces the custom `<label>` + styled `<span>` at comments-list-with-agent-toggle.html:65-76 with a proper `role="switch" aria-checked` button wrapped in a 44px hit area.

1. Canonical markup

Use a real <button role="switch"> — this is the pattern recommended by WAI-ARIA Authoring Practices for a binary toggle without a required form-submission value. Native keyboard handling (Space + Enter) is handled by the browser; no JS keyboard listener required for activation.

<!-- 44px hit area wraps the 32x16 visual so touch targets meet WCAG 2.5.5 (Level AAA) / ATAG B.2.4.1 -->
<label class="af-touch inline-flex items-center gap-2 cursor-pointer group">
  <button
    type="button"
    role="switch"
    aria-checked="false"
    aria-label="Show agent feedback"
    class="af-switch relative inline-block w-8 h-4 rounded-full bg-gray-300 dark:bg-gray-600
           data-[checked=true]:bg-teal-600 dark:data-[checked=true]:bg-teal-500
           transition-colors"
    data-checked="false"
    onclick="var b=this;var next=b.getAttribute('aria-checked')==='true'?'false':'true';b.setAttribute('aria-checked',next);b.setAttribute('data-checked',next);b.querySelector('span').style.transform=next==='true'?'translateX(16px)':'translateX(0)';"
  >
    <span aria-hidden="true"
          class="absolute top-0.5 left-0.5 w-3 h-3 rounded-full bg-white shadow
                 transition-transform"
          style="transform: translateX(0);"></span>
  </button>
  <span class="text-xs font-medium text-gray-700 dark:text-gray-300">Show agent feedback</span>
</label>

Key properties:

  • role="switch" + aria-checked — announced by AT as "switch, off" / "switch, on".
  • aria-label="Show agent feedback" — redundant with the visible text span but guarantees a label even if the surrounding <label> is stripped.
  • Native Space / Enter activate the button (browser default — no keyDown listener required).
  • focus-visible:ring-2 focus-visible:ring-teal-500 (via .af-switch:focus-visible) — canonical focus ring per focus-ring-spec.html §1.
  • 44px touch target via .af-touch wrapper — WCAG 2.5.5.
  • Color alone is not the only state signal — the thumb also slides position, and aria-checked is exposed to AT.

2. Rendered states (default, checked, focus, hover, disabled)

Default (off)

aria-checked="false"; thumb left; track gray-300

Checked (on)

aria-checked="true"; thumb right; track teal-600

Focus (keyboard)

2px teal-500 outline at 2px offset — matches canonical focus-ring spec §1

Hover

Track darkens (gray-400), soft teal halo (30% opacity), label bolder

Hover (checked)

Track darker teal (teal-700), soft teal halo

Disabled

disabled + aria-disabled="true"; opacity-50; track muted gray-200; cursor-not-allowed; still reaches 3:1 contrast for non-text UI (WCAG 1.4.11)
